## 📖 项目简介

**《十日计划》** 是一款极简主义的目标管理小程序，旨在解决长线计划容易放弃的痛点。

它不像传统的 TODO List 那样堆积任务，也不像复杂的 GTD 软件那样让人望而生畏。它的核心理念是将宏大的愿景（长卷）拆解为一个个短期的、可执行的**“十日篇章”**。

通过**新中式水墨风格**的视觉呈现，每一次打卡都如同在宣纸上盖下一枚朱砂印章，让坚持本身成为一种充满仪式感的美学体验。

## 💡 核心思想

### 1. 长卷与篇章
- **绘长卷 (Master Plan)**：定宏大愿景，续连绵墨迹。适用于 30 日以上的长线目标（如：考研、减脂、学习乐器）。书童（AI）将其裁切为若干个“十日篇章”，助你分段落笔，行稳致远。
- **裁篇章 (Quest)**：专当下修行，成十日之功。适用于单次短期的十日冲刺。书童（AI）为你定制本旬的执行细节，助你心无旁骛，速写圆满。

### 2. AI 赋能的“书童”
在这个产品中，AI 不是冷冰冰的工具，而是陪伴你成长的**“书童”**。
- **1+3 拆解法则**：书童会将你的目标拆解为 1 个每日必修的“基石任务”和 3 个循序渐进的“阶段任务”（入门/进阶/冲刺），让你在十天内既有稳固的积累，又有明显的突破。
- **动态规划**：当你完成一个篇章，书童会根据你的完成情况，为你自动生成下一个篇章的建议，让计划永远跟随你的脚步。

### 3. 水墨留白
- **盖章**：完成任务，盖上“既成”、“愿遂”的朱砂印，获得即时反馈。
- **留白**：人生无需填满，允许适度“留白”。每个篇章拥有 3 次豁免机会，跳过今日不扣分，保持心态的从容与弹性。

## 🚀 Vibe Coding 开发实践

本项目采用 **Vibe Coding** 方法论进行开发，强调人机协作的心流体验与产品美学的极致追求。

### 开发工具链
- **IDE**: [Trae](https://www.trae.ai) - 新一代 AI 原生集成开发环境。
- **AI Model**: Gemini-3-Pro-Preview - 提供强大的代码生成与逻辑推理能力。
- **Product Manager (AI)**: 
    - **Gemini**: 负责宏观架构设计、核心规则制定与技术难点攻克。
    - **豆包 (Doubao)**: 负责用户体验优化、文案润色与情感化设计。

### 开发流程
1.  **AI 结对编程**：开发者作为“驾驶员”，AI 作为“领航员”，通过对话式交互实时生成代码、修复 Bug、优化 UI。
2.  **Spec-First**: 先通过 AI 生成详细的 PRD (Product Requirement Document) 和 Spec 文档，确保需求清晰，再进行编码。
3.  **UI/UX 即时渲染**：利用 Trae 的预览功能，实时调整水墨风格的 CSS 变量与动画效果，确保视觉还原度。

## 🛠️ 技术选型

- **前端框架**: 微信小程序原生开发 (WXML, WXSS, JS)
    - 采用 CSS Variables 实现全局水墨主题配置。
    - 使用 SVG 滤镜模拟宣纸噪点纹理。
    - CSS Animation 实现印章落下、墨迹洇染等微交互。
- **后端架构** (规划中): 
    - **Runtime**: Node.js (Express)
    - **Database**: PostgreSQL
    - **AI Service**: 调用 LLM API (如 Gemini/DeepSeek) 实现目标的智能拆解。
